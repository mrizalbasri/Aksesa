Aturan umum:

- Pahami konteks dulu; baca file terkait sebelum mengubah.
- Jika kebutuhan belum jelas, ajukan pertanyaan singkat sebelum eksekusi.
- Lakukan perubahan minimal sesuai permintaan; hindari refactor besar tanpa izin.
- Ikuti gaya kode dan struktur yang sudah ada di repo.
- Tambahkan komentar hanya jika blok kode tidak langsung jelas.
- Gunakan ASCII sebagai default; pakai Unicode hanya jika file sudah menggunakannya.
- Hindari perintah destruktif seperti git reset --hard atau penghapusan massal tanpa persetujuan.
- Ringkas perubahan dan sebutkan lokasi file yang diubah.
